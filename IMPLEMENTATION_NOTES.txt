GameScope Implementation Notes
=============================

This version implements all 8 selected ideas:
1. PC Compatibility Checker (laptop search + manual specs)
2. Game Optimization Guides
3. Similar Game Recommendations
4. Game Length / Time Commitment
5. Structured Game Rating System
6. Bug / Stability Indicator
7. Value Rating
8. Player Type Recommendation

Frontend structure:
- index.html: homepage and platform pitch
- game.html: dynamic game detail page
- data.js: shared game and laptop data used by the UI
- main.js: page rendering and compatibility checker logic
- style.css: modern responsive gaming UI

Backend structure:
- backend/server.js: app entry point
- backend/routes/: auth, games, comments, contact APIs
- backend/models/: Game, User, Contact
- backend/data/seedGames.js: fallback/seed data
- backend/utils/hardware.js: hardware scoring and FPS estimation logic

How to run:
1. Open a terminal in backend/
2. Run: npm install
3. Run: npm start
4. Visit: http://localhost:4000

Notes:
- If MongoDB is not configured, the site still works in demo mode using fallback data.
- Frontend comments use localStorage for immediate demo interactivity.
- Existing individual game links now redirect to the shared dynamic game page.


Price tracker added:
- Backend route: /api/games/:slug/prices
- Auto-refresh cache loop for paid licensed games
- Optional ITAD_API_KEY + ITAD_COUNTRY in backend/.env
- Fallback mode still shows store links if live pricing is not configured
